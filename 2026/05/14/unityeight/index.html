Awake、OnEnable、Start

1 Awake() 被调用的情况可以归纳为以下三种：
  • 场景加载时：一个新的 Scene 被加载时，该场景中所有激活（Active）的 GameObject 上的脚本都会调用 Awake()。
  • 物体被激活时：如果一个原本处于非激活（Inactive）状态的 GameObject 被激活，其身上脚本的 Awake() 会立刻被调用。
  • 脚本实例化时：通过 Object.Instantiate 在运行时动态创建对象时，Awake() 会作为实例化过程的一部分被调用。
2 只要满足以下 任一条件，OnEnable() 就会被触发：
  • 场景加载且对象激活
    场景首次加载时，对于已激活的 GameObject，会先执行 Awake()，紧接着执行 OnEnable()。
  • GameObject 由非激活变为激活
    调用 SetActive(true) 后，该对象及其身上所有脚本的 OnEnable() 都会被调用。
  • 脚本组件由禁用变为启用
    将某个脚本组件的 enabled 属性设为 true（或取消勾选后重新勾选），只有该脚本的 OnEnable() 会被触发，不影响同一对象上的其他脚本。
  • 动态实例化且初始处于激活状态
    用 Instantiate 创建新物体时，如果预制体的初始状态为激活，则新实例会依次调用 Awake() 和 OnEnable()。

关键特性：可重复调用
与 Awake 在整个生命周期只执行一次不同，OnEnable 在每次启用时都会被调用。只要对象/组件禁用了再启用，它就会再次执行。

3 Start 会在脚本首次启用后的 Update 之前被调用，并且只会执行一次。
  • 在第一帧Update之前调用（仅一次）
  • 适合其他组件完成初始化后的设置

例如，根据另一个脚本中初始化的变量来设置自己的初始状态。

Update()

  • 每帧调用一次（不固定时间间隔）
  • 适合游戏逻辑、输入处理

LateUpdate()

  • 在所有Update之后调用
  • 适合摄像机跟随等需要最后调整的逻辑

OnDestroy()

  • 对象被销毁前调用
  • 适合释放资源、清理引用

Unity 协程、线程与进程的区别

  • 进程：操作系统分配资源的基本单元，一个进程可包含多个线程。
  • 线程：CPU 调度的基本单元，同一进程内的线程共享内存资源。
  • Unity 线程模型：大部分 Unity API 和脚本逻辑运行在主线程，底层渲染、Job System、Burst 等会利用工作线程，因此 Unity 并非完全单线程，但脚本默认在主线程执行。
  • 协程（Coroutine）：运行在主线程上的分时机制，不创建新线程。协程在 yield return 时挂起、条件满足时恢复，适合把耗时任务拆成多帧或定时执行。
  • 协程特点：
    • 不涉及线程同步或加锁问题，能直接访问 Unity 主线程 API。
    • 同一时间只会执行某个协程片段，适合分步或定时逻辑。
    • 不是并行线程，无法利用多核并行。

c#中能否多继承父类

在 C# 中，不可以多继承（即一个类同时继承多个父类）。C# 只支持单继承——一个类只能有一个直接基类。

不过，C# 通过其他机制提供了多继承的部分能力：

  • 接口多实现：一个类可以实现多个接口，从而获得多种“契约”行为。
  • 默认接口方法（C# 8.0+）：接口可以包含默认实现，进一步缩小了与多重继承的差距。
  • 组合/依赖注入：通过包含其他类的实例来复用功能，这是更推荐的做法。
